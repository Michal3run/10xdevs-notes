# m2l4 — Key Takeaways

- Context drift to **przewidywalna własność systemu**, nie zły dzień modelu. Sygnał: agent powtarza fix-it loopy, miesza nazwy z różnych slice'ów, „pamięta" decyzje, których nie podjęliście. Reakcja: świeża sesja, wczytaj `plan.md` + `## Progress`, kontynuuj fazę. Nie „popraw to jeszcze raz". Lost-in-the-middle z architektury transformerów potwierdza Chroma "Context Rot" (2025).
- External research dziel na dwa narzędzia: **Exa** (agentic search, najwyższa ocena na Technical Documentation w benchmarku AIMultiple 05/2026) wybiera kandydata; **Context7** (`resolve-library-id` → `query-docs`) wstrzykuje aktualne API zwycięzcy. Bez Context7 agent halucynuje funkcje sprzed dwóch wersji.
- Internal `/10x-research` zapisuje **wyłącznie** `research.md` w folderze change'a — to celowy bezpiecznik. Każde znalezisko musi przyjść z `file:line`. Brak referencji = brak claimu, nie szacunek. Skill nie wybierze ci biblioteki; jeśli zapytasz, zaproponuje własną implementację SRS, co jest „wzorcem na piasku".
- `/10x-frame` to koło ratunkowe, nie część głównego workflow. Włączasz, gdy mimo prób kontynuacji rozmowy poprawki nie naprawiają niczego (więcej niż jedna sesja). Frame Brief rozdziela **observation / cause / fix** — najczęstszy bug to fix odziedziczony z poprzedniego ticketa, choć przyczyna jest gdzie indziej. Research adresuje brak danych, frame adresuje błędne nazwanie problemu.
- Wzorzec docs agent-friendly (`llms.txt`, Cloudflare `Accept: text/markdown` + nagłówek `x-markdown-tokens`, Vercel `.md` suffix) to dziś realny filtr wyboru bibliotek. Resend → 500k devów w rok i 63% propozycji Claude Code'a "dodaj wysyłkę maili" przez agent-first docs. Patrz na to przy researchu, zanim wybierzesz tooling.
