# m2l3 — Linki

## Tools

- [Claude Code Review documentation](https://code.claude.com/docs/en/code-review) — wbudowany system code review w Claude Code; warto porównać z `/10x-impl-review` (gdzie raport ląduje w pliku, gdzie scorecard, jak wygląda triage).

## Reference

- [Google Engineering Practices: What to look for in a code review](https://google.github.io/eng-practices/review/reviewer/looking-for.html) — kanoniczna check-lista (design, funkcjonalność, złożoność, testy, nazwy, komentarze).
- [Expectations, Outcomes, and Challenges of Modern Code Review (Bird, Bacchelli, MSR/ICSE 2013)](https://www.microsoft.com/en-us/research/publication/expectations-outcomes-and-challenges-of-modern-code-review/) — review jako transfer wiedzy i zrozumienie zmiany, nie tylko łapanie defektów.
- [Do Users Write More Insecure Code with AI Assistants? (Perry et al., CCS 2023)](https://arxiv.org/abs/2211.03622) — fałszywa pewność przy AI-generated code; uzasadnienie dla Safety & Quality jako osobnego wymiaru scorecarda.
- [SWE-PRBench (Kumar 2026)](https://arxiv.org/abs/2603.26130) — AI review wciąż ustępuje ekspertowi; strategia kontekstu waży więcej niż jego ilość.

## Prework

- Prework "approve bez obrony" — antywzorzec „AI dowozi wynik, klikam akceptację" jest punktem startowym całej dyscypliny scorecard + triage. Lekcja odwołuje się do niego bez numeru — sięgnij na platformę kursową.
