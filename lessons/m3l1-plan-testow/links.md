# m3l1 — Linki

## Tools

- [Repomix](https://repomix.com/) — pakuje repo jako brief dla LLM-a; przydaje się, gdy chcesz wstępnie sprofilować strukturę przed test-planem.
- [GitIngest](https://gitingest.com/) — lekka alternatywa do Repomixa; tekstowy kontekst repo do wklejenia w sesję planowania.

## Reference

- [ISTQB: Risk-Based Testing](https://glossary.istqb.org/en_US/term/risk-based-testing) — kanoniczny słownik dla impact × likelihood; kotwica, że ta zasada nie powstała pod LLM-y.
- [OWASP API Security Top 10 — BOLA (IDOR)](https://owasp.org/API-Security/editions/2023/en/0xa1-broken-object-level-authorization/) — podstawa dla wiersza autoryzacji w mapie ryzyk; klasy nadużyć, które nie wypłyną z user stories.

- [Prework 1.3 — Jak uczyć się i rozwijać z AI](https://platforma.przeprogramowani.pl/external/10xdevs-3-prework/pl/03) — przypomnienie, że mapy ryzyk nie akceptujemy bez obrony decyzji.
- [Prework 3.2 — Wzorce i antywzorce promptowania](https://platforma.przeprogramowani.pl/external/10xdevs-3-prework/pl/10) — prompt jako kontrakt; tło dla `test-plan.md` jako trwałego planu jakości.
